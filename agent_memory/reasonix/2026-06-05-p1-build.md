## ❌ 失误与教训
- **TFM 不兼容**：测试项目用了 net8.0 而非 net8.0-windows，与主项目冲突。教训：检查 csproj TargetFramework 一致性。
- **MainWindow.xaml StatusBar 在 Grid 外**：Window 只能有一个直接子级。教训：先确认容器结构再放元素。
- **CliResult 重复定义**：在 CliRunner.cs 中又定义了一遍，与独立文件 CliResult.cs 冲突。教训：优先复用已有类型。
- **ProjectView.xaml ElementName=Root 缺少 x:Name**：DataTemplate 内跨层绑定需要在顶层元素声明名称。教训：用 ElementName 前确认目标存在。
- **RadioButton 绑定设计错误**：字符串属性无法直接用 IsChecked 绑定两个 RadioButton。教训：RadioButton 多选一需要 Bool 属性 + Converter。
- **命令行 Prompt 直接拼接**：多行中文作为命令行参数极容易出转义问题。教训：大文本用临时文件传递。

## 🔧 改进建议
1. 写代码前先 `dotnet build`，确认当前状态无编译错误
2. 新增文件前用 `grep` 检查是否有同名类已存在
3. 所有 XAML 绑定写完后，逐个对照 ViewModel 属性确认名称存在
4. CLI 命令永远不要直接拼接用户文本，写临时文件 + `--file` 参数
